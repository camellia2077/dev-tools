## v0.1.0 - 2025-01-18

### 架构重构 (Architecture Refactoring)

* **模块解耦与单一职责 (SRP)**: 将原有的单体脚本拆解为高内聚的独立模块：
* `scanner`: 专注于文件系统遍历 (IO)。
* `parser`: 专注于正则表达式解析 (Core Logic)。
* `classifier`: 封装头文件分类策略 (Strategy Pattern)。
* `analyzer`: 负责数据的聚合、排序与报告生成 (Core Logic)。
* `writer`: 专注于流式输出与格式化 (IO)。


* **逻辑分层**: 建立了 `core/` (纯逻辑层) 与 `io/` (基础设施层) 的目录结构，实现了业务核心与外部交互的物理隔离。
* **依赖注入 (Dependency Injection)**: 重构 `main.py` 作为组合根 (Composition Root)，通过构造函数注入配置与策略，移除了模块间的硬编码依赖，提升了可测试性。

### 新增特性 (Features)

* **多级排序策略**: 重新设计了头文件的排序算法，采用三级排序键以优化 PCH 的命中率与可读性：
1. **顶级目录** (字母升序)：确保同一模块的头文件物理相邻。
2. **使用次数** (数值降序)：同一模块内，高频头文件优先。
3. **文件路径** (字母升序)：作为相同频率下的稳定排序依据。


* **视觉分组**: 输出生成器 (`writer`) 新增逻辑，在检测到顶级目录变更时自动插入空行，显著提升生成的 PCH 源码可读性。

### 性能与健壮性 (Performance & Stability)

* **流式扫描**: `FileFinder` 采用 Python 生成器 (`yield`) 实现，避免一次性将所有文件路径加载到内存中，优化了在大规模代码库下的内存占用。
* **编码容错**: 文件读取操作增加 `utf-8` 编码指定及 `errors='ignore'` 处理，防止因个别源代码文件编码异常导致工具崩溃。
